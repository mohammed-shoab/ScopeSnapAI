📱 Mobile Responsive Design

Mobile Layout — Tailwind md: breakpoints at 768px

All pages use Tailwind CSS responsive classes. On screens below 768px, the layout switches to:


Sidebar — collapses to a top bar with a hamburger menu (md:hidden / md:block classes in SidebarNav.tsx)
Main content — shifts from md:ml-60 (240px sidebar offset) to full-width with pt-16 top padding for the mobile header
Stat cards — stack from grid-cols-4 to single column
2-col sections — stack from grid-cols-2 to single column
Assess page — photo upload area goes full width, designed mobile-first for on-site tech use


⚠️ Note: Mobile screenshots couldn't be captured in this session — the browser environment runs on a 1920px display and Chrome's minimum window size prevents actual viewport resizing to mobile widths. To view the mobile layout, open http://localhost:3000 in Chrome DevTools → Toggle device toolbar → set to iPhone 14 (390×844).

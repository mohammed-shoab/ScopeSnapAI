Bug Tracker — All 7 Resolved
FIXED S1
TypeError: estimates.filter is not a function — Dashboard
Dashboard crashed on load. Stale .next cache + unguarded filter call on non-array API response.
✓ Fixed Session 1: Array.isArray() guards. Cleared .next cache via Alpine Docker container.
FIXED S1
TypeError: Cannot read properties of undefined (reading 'map') — Billing page
Billing page crashed when dev user not in billing system. API error stored as valid subscription.
✓ Fixed Session 1: Added r.ok guard + !subData.detail check + sub.features ?? [] fallback.
FIXED S1
All pages unstyled — postcss.config.js missing
All pages rendered without Tailwind CSS — raw HTML only.
✓ Fixed Session 1: Created scopesnap-web/postcss.config.js.
FIXED S3
"Saves $[object Object]/yr" — Estimate Builder
energy_savings field in API response is a dict {annual_savings: 420, five_year_savings: 2100, ...}, not a number. Template called .toLocaleString() directly on the object.
✓ Fixed Session 3: Added EnergySavings interface. Type guard extracts annual_savings from object vs. treats as number. File: app/(app)/estimate/[id]/page.tsx
FIXED S3
Infinite loop — Markup PATCH API hangs on every slider change
In PATCH /api/estimates/{id}, updated_options.append(option) was inside for option in updated_options. Python's for loop iterates over a growing list, causing infinite growth and a hung request.
✓ Fixed Session 3: Removed the errant .append() call. Options are mutated in-place on the deepcopy. File: scopesnap-api/api/estimates.py
FIXED S3
"Failed to fetch" — Generate Documents crashes ASGI connection
WeasyPrint import failure (import weasyprint at module level) was closing the TCP connection without sending any HTTP response. Browser received no response, showing "Failed to fetch". Also blocked the Send tab entirely.
✓ Fixed Session 3: Lazy try/except import. If WeasyPrint unavailable, endpoint returns HTTP 200 with pdf_warning field. homeowner_report_url is always set so Send flow is never blocked. File: scopesnap-api/api/estimates.py
⚠ Note: Actual PDF generation may still produce a placeholder in Docker if libpango/libcairo not installed. Run docker compose exec api pip show weasyprint to verify. The flow is unblocked either way.
FIXED S3
Analytics page "API error 404" — misleading error message
The 404 was "User not found" (user wasn't provisioned yet). Error card always showed "Only owners and admins" regardless of actual error type.
✓ Fixed Session 3: Error card now conditionally shows auth message only for actual 403/auth errors. User test_user_mike now provisioned as owner. File: app/(app)/analytics/page.tsx
